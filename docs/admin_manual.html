환경변수 관리

Railway 대시보드 → BizModelCanvas 프로젝트 → 웹앱 서비스 → Variables 탭에서 관리합니다.

필수 환경변수

변수명설명예시값
SECRET_KEYFlask 세션 암호화 키 (랜덤 문자열)abc123xyz... (32자 이상 권장)
ANTHROPIC_API_KEYClaude AI API 키sk-ant-api03-...
DATABASE_URLPostgreSQL 연결 문자열 (Railway 자동 주입)postgresql://user:pass@host/db
SENDGRID_API_KEYSendGrid 이메일 발송 API 키SG.xxxxxxxxxxxxxxx
MAIL_DEFAULT_SENDER발신자 이메일 주소heoksgg@gmail.com

선택 환경변수 (기본값 사용 가능)

변수명기본값설명
MAIL_SERVERsmtp.gmail.comSMTP 서버 (현재 미사용)
MAIL_PORT587SMTP 포트 (현재 미사용)
MAIL_USE_TLStrueTLS 사용 여부 (현재 미사용)
MAIL_USERNAMESMTP 계정 (현재 미사용)
MAIL_PASSWORDSMTP 비밀번호 (현재 미사용)
SECRET_KEY는 반드시 안전한 랜덤 문자열로 설정해야 합니다. 기본값(dev-secret-key) 사용 시 보안 취약점이 발생합니다.

환경변수 변경 시 주의사항

환경변수 저장 후 Railway가 자동으로 재배포합니다. 재배포 중에는 약 1~2분간 서비스가 중단될 수 있습니다.
03

회원 관리

관리자 메뉴(/admin/)에서 모든 회원을 관리합니다. 관리자 역할을 가진 계정만 접근 가능합니다.

회원 상태 종류

상태설명로그인 가능 여부
대기 pending가입 후 승인 대기 중❌ 불가
활성 active정상 사용 가능✅ 가능
비활성 inactive계정 정지 상태❌ 불가

회원 역할 종류

역할설명관리자 메뉴 접근
관리자 admin전체 관리 권한 보유✅ 가능
일반회원 member일반 사용자❌ 불가

신규 회원 승인

1
관리자 메뉴 접속사이드바의 '⚙️ 관리자' 메뉴를 클릭합니다.
2
승인 대기 목록 확인상단 통계에서 '대기' 숫자를 확인하고, 승인 대기 목록에서 회원 정보를 검토합니다.
3
승인 또는 거절승인: 상태가 'active'로 변경되어 즉시 로그인 가능합니다. 거절: 상태가 'inactive'로 변경됩니다.
신규 가입 알림 기능은 현재 미구현 상태입니다. 정기적으로 관리자 페이지를 확인하여 대기 회원을 처리해주세요.

회원 상태·역할 변경

전체 회원 목록에서 각 회원의 상태와 역할을 변경할 수 있습니다.

작업방법제한사항
상태 변경회원 목록 → 상태 버튼 클릭자기 자신의 상태 변경 불가
역할 변경회원 목록 → 역할 토글 버튼자기 자신의 역할 변경 불가

회원 정보 수정 및 삭제

회원 삭제 시 해당 회원의 모든 프로젝트와 분석 데이터가 함께 삭제됩니다. 삭제 전 반드시 확인하세요.
작업방법수정 가능 항목
정보 수정회원 목록 → 수정 버튼이름, 이메일, 조직, 직책, 비밀번호, 역할, 상태
회원 삭제회원 상세 → 삭제 버튼— (복구 불가)
04

데이터베이스 관리

DB 접속 방법

Railway 대시보드 → PostgreSQL 서비스 → Query 탭에서 직접 SQL 실행 가능합니다.

주요 테이블 구조

테이블설명주요 컬럼
users사용자 계정id, username, email, role, status, reset_token
project프로젝트 정보id, name, organization, owner_id, is_public
analysis모듈 분석 데이터id, project_id, module_type, input_data(JSON), ai_result
qn_a질의응답 게시글id, owner_id, title, content, ai_answer, ai_pending
qn_a_comment질의응답 댓글id, qna_id, owner_id, content, ai_answer

컬럼 추가 시 주의사항

이 프로젝트는 Flask-Migrate를 사용하지 않습니다. 모델에 컬럼을 추가한 경우 Railway PostgreSQL Query 탭에서 직접 실행해야 합니다.

예시: users 테이블에 phone 컬럼 추가
ALTER TABLE users ADD COLUMN phone VARCHAR(20);

데이터 초기화 (주의)

아래 SQL은 모든 사용자 데이터를 영구적으로 삭제합니다. 신중하게 실행하세요.
전체 데이터 초기화 순서 (외래키 순서 주의):
TRUNCATE TABLE qn_a_comment, qn_a, analysis, project, users RESTART IDENTITY CASCADE;

백업

Railway PostgreSQL 서비스 → Backups 탭에서 수동 백업을 생성할 수 있습니다. 중요한 작업 전에는 반드시 백업을 생성하세요.

롤백 방법

Railway 대시보드 → Deployments 탭 → 이전 배포 항목 → Rollback 버튼을 클릭합니다.

롤백 시 코드는 이전 버전으로 돌아가지만 데이터베이스 스키마는 롤백되지 않습니다. DB 마이그레이션이 포함된 경우 별도 처리가 필요합니다.

로컬 개발 환경 실행

1. 저장소 클론: git clone https://github.com/heoks99/BizModelCanvas.git
2. 패키지 설치: pip install -r requirements.txt
3. .env 파일 설정 (.env.example 참고)
4. 서버 실행: python run.py → http://localhost:5000

주요 장애 유형 및 대응

증상원인조치
로그인 후 세션이 즉시 만료됨SECRET_KEY 미설정 또는 재시작마다 변경Railway Variables에 고정 SECRET_KEY 설정
AI 분석 실행 안 됨ANTHROPIC_API_KEY 미설정 또는 만료/api/health 확인, API 키 재발급 후 Variables 업데이트
이메일 발송 안 됨SENDGRID_API_KEY 미설정, 발신자 미인증/api/smtp-check 확인, SendGrid Sender Authentication 확인
PDF 생성 500 에러프로젝트 필드 None 값, 폰트 파일 누락에러 로그 확인, 폰트 파일(app/static/fonts/) 존재 여부 확인
데이터가 사라짐SQLite 사용 중 컨테이너 재시작PostgreSQL 연결 확인, DATABASE_URL 환경변수 확인
Internal Server Error코드 오류 또는 DB 연결 실패Railway Logs 탭에서 상세 에러 확인

API 키 관리

API 키는 절대 GitHub에 커밋하지 마세요.
.env 파일은 .gitignore에 포함되어 있습니다. 확인 명령: git check-ignore -v .env
API 키만료/갱신 주기갱신 방법
ANTHROPIC_API_KEY만료 없음 (수동 폐기 가능)console.anthropic.com에서 재발급 후 Railway Variables 업데이트
SENDGRID_API_KEY만료 없음 (수동 폐기 가능)app.sendgrid.com → API Keys에서 재발급

접근 제어

  • 모든 페이지는 로그인 필수 (@login_required)
  • 관리자 페이지는 admin 역할 필수 (@admin_required)
  • 프로젝트 편집은 소유자만 가능
  • 진단 엔드포인트 /api/smtp-check는 로그인 없이 접근 가능 — 운영 안정화 후 @login_required 추가 권장
